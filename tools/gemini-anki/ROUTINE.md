# Anki 장기기억 루틴

Gemini와 공부한 내용을 형식 걱정 없이 쌓아두면, AI가 핵심만 뽑아 Anki 카드로
만들어주고, 간격 반복(SRS)으로 장기 기억에 남게 하는 개인 루틴.

## 왜 이렇게 만들었나

- **노이즈 줄이기**: Google Takeout으로 대화 기록 전체를 내보내면 관련 없는
  내용까지 다 섞여 나온다. 대신 공부하다가 남기고 싶은 부분만 그때그때
  파일에 쌓는 방식(`study_log_*.md`)을 기본으로 한다.
- **형식 안 갖춰도 됨**: `--ai` 옵션으로 Gemini API가 직접 긴 글에서 핵심
  표현/개념을 뽑아준다. 정리된 글이든 편하게 쓴 글이든 상관없다.
- **ADHD 친화적 구조**: 주제별로 덱을 나눠서 "오늘은 중국어만 5분" 처럼
  작고 구체적인 단위로 시작할 수 있게 했다. 새 카드 개수는 일부러
  제한하지 않는다 (원하는 대로 쌓아도 됨).
- **반복 = 중요 신호**: 같은 표현을 또 올리면 새 카드를 만드는 대신, 그
  카드의 다음 복습일을 오늘로 당긴다. 자꾸 다시 언급한다는 건 그만큼
  중요하다는 뜻으로 보고 복습 빈도를 자동으로 높이는 것.

## 전체 구조

```
공부하다 남기고 싶은 내용
  → study_log_*.md 에 그냥 붙여넣고 저장 (형식 무관, 한국어/영어/중국어 다 됨)
  → run_study_log.bat 더블클릭
      → Gemini API가 핵심 표현/뜻을 자동 추출
      → 새 표현 → Anki 카드로 추가
      → 이미 있던 표현 → 복습일을 오늘로 당김 (더 자주 보게)
  → PC Anki에서 동기화(Sync)
  → 폰(AnkiDroid)에서 아무 때나 복습
```

## 주제별 파일 ↔ 덱

| 로그 파일 | Anki 덱 |
|---|---|
| `study_log_chinese.md` | 중국어 |
| `study_log_english.md` | 영어 |
| `study_log_bi.md` | BI |
| `study_log_python.md` | 파이썬 |

필요한 주제만 파일로 만들어 쓰면 된다 (4개 다 만들 필요 없음). 새 주제를
추가하고 싶으면 `run_all_logs.py`의 `LOG_DECK_MAP`에 한 줄만 추가한다.

## 하루/주간 루틴

1. **공부할 때**: Gemini와 대화하거나 책/자료를 보다가 남기고 싶은 내용이
   생기면, 해당 주제 `.md` 파일에 그냥 붙여넣고 저장. 형식 신경 안 써도 됨.
2. **카드 만들 때** (하루 끝, 또는 생각날 때): Anki 켜두고
   `run_study_log.bat` 더블클릭 — 존재하는 로그 파일들을 한 번에 처리.
3. **동기화**: PC Anki에서 Sync 버튼 → 폰에도 반영.
4. **복습**: 폰이든 PC든 아무 때나 AnkiDroid로 카드 복습. **"다시(Again)"를
   정직하게 누르는 게 핵심** — Anki가 그걸 기준으로 복습 간격을 조절해서
   장기 기억에 남긴다.
5. **가끔 (주 1회 정도)**: 자주 까먹는 카드 확인.
   ```
   python find_forgotten_cards.py --print-prompt
   ```
   여기서 나온 프롬프트를 Gemini에 붙여넣어 다시 설명을 받고, 그 내용을
   다시 `.md` 파일에 추가 → `run_study_log.bat` 재실행. 이러면 "자꾸 까먹는
   것 위주로 다시 배우기" 루프가 완성된다.

## 설정값 메모

- 새 카드 개수 제한: **없음** (Anki 기본값 그대로, 원하는 만큼 쌓기)
- 중복 표현 처리: 새로 안 만들고 **오늘 복습으로 당김**
- Anki 덱 나누는 이유: ADHD 특성상 "오늘은 이것만" 하고 작게 시작하기 쉬우라고
- AI 모델: `gemini-flash-latest` (Google이 모델명을 자주 바꾸므로, 오류 나면
  `--model` 옵션으로 다른 모델명 지정 가능)

## 참고 파일

- `README.md` — 설치/설정 전체 가이드 (Anki, AnkiConnect, API 키 등)
- `run_study_log.bat` — 더블클릭 실행 파일 (한글 없음, 순수 실행 트리거)
- `run_all_logs.py` — 실제 로직 (파일↔덱 매핑, 안내 메시지)
- `find_forgotten_cards.py` — 자주 까먹는 카드 찾기
